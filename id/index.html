美洽 widget chat tidak muncul / perbaiki error integrasi

Sudah menempel kode 美洽(Meiqia) tapi widget chat web tidak muncul, tombol chat tak pernah tampil, konsol melaporkan meiqia.js diblokir, atau agen tidak menerima chat? Perjelas tiga hal dulu: apakah skrip tidak dimuat sama sekali (penempatan / adblock / cache), dimuat tapi tertutup gaya / penumpukan, atau dimuat baik tapi konfigurasi tidak cocok (entId / domain / siklus hidup SPA)? Cari gejala atau kata kunci (mis. not showing, adblock, entId, SPA, mobile, sdk push): pertama perilaku L1 & posisi resmi, lalu akar masalah L2 & solusi. Pemasangan / panduan? Lihat unduh 美洽 / panduan integrasi layanan pelanggan.

Langkah 1: periksa apakah skrip dimuat, lalu bagi jadi empat kelompok

Saat chat 美洽 tidak muncul, 9 dari 10 kali itu salah satu dari empat kelompok; lihat dulu apakah skrip benar-benar dimuat: F12 → Network, cari meiqia.js; tanpa permintaan atau status bukan 200 berarti tidak dimuat. ① Gagal muat: kode di tempat salah (harus sebelum </body>, bukan di <head>), diblokir aturan adblock (ERR_BLOCKED_BY_CLIENT), tertahan cache CDN / browser, atau sertifikat HTTPS / konten campuran. ② Konfigurasi / otorisasi: entId tidak cocok dengan workbench (dimuat tapi agen tidak menerima chat), atau domain situs belum diotorisasi di konsol. ③ Integrasi framework: SPA Vue/React/Next menghancurkan kontainer saat ganti rute dan widget tidak dibangun ulang. ④ Tampilan / seluler / SDK: CSS situs mendorong gelembung ke luar layar, plugin pihak ketiga menumpuk DOM, seluler perlu deploy sendiri, in-app perlu SDK. Lebih lengkap: panduan integrasi web 美洽.

Gagal muatpenempatan · adblock · cache · HTTPS
Tempel JS 美洽 sebelum (bukan di ); F12 → Network, cari meiqia.js dan periksa status (harus 200); ERR_BLOCKED_BY_CLIENT biasanya ekstensi adblock — uji di mode penyamaran atau whitelist; bersihkan cache CDN setelah publikasi; pastikan HTTPS penuh tanpa konten campuran.
Konfigurasi / otorisasientId · whitelist domain · sub-kanal
entId adalah id unik perusahaan Anda; jika tidak cocok dengan workbench «skrip dimuat tapi agen tidak menerima chat» (Pengaturan - Tim - cari ID); domain situs harus diotorisasi via «Tambah situs integrasi»; pakai sub-kanal (probe) per lini bisnis.
Integrasi frameworkSPA · ganti rute · manualInit
SPA Vue/React/Next menghancurkan DOM saat ganti rute dan widget tidak dibangun ulang otomatis; pakai _MEIQIA('manualInit') untuk menghentikan auto-init dan panggil _MEIQIA('init') di hook rute (useEffect / mounted) untuk memasang ulang.
Masalah tampilankonflik gaya · penumpukan · plugin pihak ketiga
Dimuat tapi tak terlihat: biasanya CSS global situs menimpa posisi gelembung, atau plugin heatmap / analitik / SEO mengubah DOM dan menutup kontainer; F12 → Elements, cari kontainer meiqia dan periksa display:none, z-index kalah, atau koordinat luar layar.
Seluler / SDKweb seluler · AppKey · push
Web seluler / PC adalah snippet yang sama tapi perlu deploy sendiri; in-app pakai SDK native (dulu «Tambah konfig APP» di konsol untuk AppKey); push terbagi «tanpa push» (hanya in-app) dan «server push kustom» (terkirim bahkan setelah keluar app).
Panggilan APIwithoutBtn · showPanel · info pelanggan
Tombol sendiri? _MEIQIA('withoutBtn') menyembunyikan tombol native, dan panggil _MEIQIA('showPanel') saat klik untuk membuka chat; kirim / sinkron info pelanggan dalam waktu init, jika tidak takkan berlaku.

Langkah 2: akar masalah yang paling sering terlewat → L2 (widget = injeksi JS asinkron pihak ketiga lintas domain)

Satu gagasan cukup: widget web 美洽 bukan komponen statis yang tertanam di halaman Anda — ia adalah meiqia.js yang dimuat secara asinkron dari domain eksternal 美洽 lalu secara dinamis menyuntikkan kontainer chat (DOM / iframe) dan membuka koneksi lintas-asal yang bertahan lama ke server 美洽. Ini menjelaskan sebagian besar «hilang misterius»: ① ia skrip pihak ketiga lintas domain, jadi AdBlock / uBlock memblokirnya lewat daftar «pelacakan / iklan» (ERR_BLOCKED_BY_CLIENT) dan tombol tak pernah muncul (padahal konsol tampak baik); ② ia menyuntik DOM secara asinkron: di <head> ia memblokir, perubahan rute SPA menghancurkan kontainer dan tidak dibangun ulang otomatis (pakai manualInit + _MEIQIA('init')), dan plugin heatmap / analitik yang mengubah DOM bisa menutupinya; ③ ia terikat ke perusahaan Anda via entId dan digerbangi whitelist domain: dengan entId salah atau domain tak diotorisasi skrip dimuat tapi «tak menyambung chat»; ④ seluler dan PC adalah dua integrasi terpisah, in-app adalah SDK (AppKey), dan push terbagi «tanpa push» vs «server push kustom». Setelah Anda melihat rantai injeksi ini, setiap gejala di bawah punya satu logika. Di bawah panel swa-periksa; lebih bawah lima metode dan perkiraan 2026.

美洽 sebab tidak muncul & perbandingan metode integrasi (perkiraan 2026)

Berikut adalah perkiraan 2026 yang dirangkum dari bantuan resmi 美洽 (Kanal akses / API widget web JavaScript) dan troubleshooting integrasi publik (bukan komitmen vendor maupun pengukuran langsung; sebagai acuan, berubah menurut versi dan kebijakan browser):

Apa sebenarnya integrasi ituwidget web = JS asinkron pihak ketiga lintas domain yang menyuntik DOM + koneksi lintas-asal bertahan lama (bukan komponen statis tertanam); maka terpengaruh penempatan, aturan adblock, penumpukan CSS, siklus hidup SPA

Basis perkiraan: garis dasar sumber + ekstrapolasi waktu (meiqia.com/help Kanal akses / widget web JavaScript, panduan integrasi meiqia.im, troubleshooting publik, 2026); berubah menurut versi dan kebijakan pemblokiran browser. Ikuti info resmi 美洽 terbaru. Tidak resmi · lokalisasi LLM.

Kasus nyata — baca cepat

  • Kode ditempel tapi tak ada gelembung chat: pertama F12 → Network, cari meiqia.js — tanpa permintaan atau bukan-200 berarti salah tempat / tertahan cache; bersihkan cache CDN atau buka ulang di penyamaran.
  • Konsol bilang meiqia.js net::ERR_BLOCKED_BY_CLIENT: adblock (AdBlock / uBlock / AdGuard) memblokir skrip chat pihak ketiga lewat aturan iklan — matikan ekstensi atau whitelist situs; ini juga sebab «hilang bagi sebagian pengguna padahal konsol Anda baik».
  • Skrip dimuat (konsol typeof _MEIQIA adalah function) tapi tak ada tombol: biasanya CSS tema mendorong gelembung ke luar layar atau plugin heatmap / analitik mengubah penumpukan DOM — F12 → Elements, cari kontainer 美洽 dan periksa satu per satu.
  • Beranda punya chat di SPA Vue / React, tapi rute kedua tidak: ganti rute SPA menghancurkan kontainer dan widget tak dibangun ulang — pakai manualInit untuk menghentikan auto-init dan panggil _MEIQIA('init') di hook rute untuk memasang ulang.
  • Chat terbuka tapi agen tidak menerima pesan pengunjung: kemungkinan besar entId tidak cocok dengan workbench (menyalin snippet orang lain / akun tercampur) — bandingkan ID perusahaan di Pengaturan - Tim - cari ID dengan entId di kode.
  • Jalan di staging, tak dimuat di domain produksi: domain produksi belum diotorisasi via «Tambah situs integrasi» — tambahkan domain live ke daftar lalu publikasikan ulang.
  • Ingin membuang tombol bundar native 美洽 dan pakai tombol «Hubungi» sendiri: panggil _MEIQIA('withoutBtn') untuk menyembunyikan native, lalu ikat _MEIQIA('showPanel') ke tombol Anda untuk membuka chat.

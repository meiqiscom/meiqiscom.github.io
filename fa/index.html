美洽 ابزارک گفتگو نمایش داده نمی‌شود / رفع خطای یکپارچه‌سازی

کد 美洽(Meiqia) را چسبانده‌اید اما ابزارک گفتگوی وب نمایش داده نمی‌شود، دکمه گفتگو هرگز ظاهر نمی‌شود، کنسول گزارش می‌دهد meiqia.js مسدود است، یا کارشناسان گفتگو دریافت نمی‌کنند؟ ابتدا سه چیز را روشن کنید: آیا اسکریپت اصلاً بارگذاری نشده (مکان / adblock / حافظه پنهان)، بارگذاری شده اما با استایل / انباشت پنهان شده، یا خوب بارگذاری شده اما پیکربندی همخوان نیست (entId / دامنه / چرخه عمر SPA)؟ علامت یا کلیدواژه خود را جستجو کنید (مثل not showing، adblock، entId، SPA، mobile، sdk push): ابتدا رفتار L1 و موقعیت رسمی، سپس علت ریشه‌ای L2 و راه‌حل. نصب / راهنما؟ ببینید دانلود 美洽 / راهنمای یکپارچه‌سازی پشتیبانی مشتری.

گام ۱: بررسی کنید اسکریپت بارگذاری شده یا نه، سپس به چهار گروه تقسیم کنید

وقتی گفتگوی 美洽 نمایش داده نمی‌شود، در ۹ از ۱۰ بار یکی از چهار گروه است؛ ابتدا ببینید آیا اسکریپت اصلاً بارگذاری شده: F12 → Network، meiqia.js را جستجو کنید؛ نبود درخواست یا وضعیت غیر ۲۰۰ یعنی بارگذاری نشده. ① شکست بارگذاری: کد در جای اشتباه (باید پیش از </body> باشد، نه در <head>)، با قانون adblock مسدود (ERR_BLOCKED_BY_CLIENT)، گیرافتاده در حافظه پنهان CDN / مرورگر، یا گواهی HTTPS / محتوای مختلط. ② پیکربندی / مجوز: entId با میز کار همخوان نیست (بارگذاری می‌شود اما کارشناسان گفتگو دریافت نمی‌کنند)، یا دامنه سایت در کنسول مجاز نشده. ③ یکپارچه‌سازی فریم‌ورک: SPA با Vue/React/Next هنگام تغییر مسیر ظرف را نابود می‌کند و ابزارک بازسازی نمی‌شود. ④ نمایش / موبایل / SDK: CSS سایت حباب را بیرون صفحه می‌راند، افزونه شخص ثالث روی DOM می‌نشیند، موبایل استقرار جدا می‌خواهد، درون‌برنامه SDK می‌خواهد. کامل‌تر: راهنمای یکپارچه‌سازی وب 美洽.

شکست بارگذاریمکان · adblock · حافظه پنهان · HTTPS
JS مربوط به 美洽 را پیش از بچسبانید (نه در )؛ F12 → Network، meiqia.js را جستجو و وضعیت را بررسی کنید (باید ۲۰۰ باشد)؛ ERR_BLOCKED_BY_CLIENT معمولاً افزونه adblock است — در حالت ناشناس آزمایش یا به فهرست سفید بیفزایید؛ پس از انتشار حافظه پنهان CDN را پاک کنید؛ HTTPS کامل بدون محتوای مختلط را تضمین کنید.
پیکربندی / مجوزentId · فهرست سفید دامنه · زیرکانال
entId شناسه یکتای شرکت شماست؛ اگر با میز کار همخوان نباشد «اسکریپت بارگذاری می‌شود اما کارشناسان گفتگو دریافت نمی‌کنند» (تنظیمات - تیم - جستجوی شناسه)؛ دامنه سایت باید از طریق «افزودن سایت یکپارچه‌سازی» مجاز شود؛ برای هر خط کسب‌وکار زیرکانال (پروب) جدا به‌کار ببرید.
یکپارچه‌سازی فریم‌ورکSPA · تغییر مسیر · manualInit
SPA با Vue/React/Next هنگام تغییر مسیر DOM را نابود می‌کند و ابزارک بازسازی نمی‌شود؛ برای توقف مقداردهی خودکار از _MEIQIA('manualInit') استفاده کنید و برای نصب مجدد در یک قلاب مسیر (useEffect / mounted) _MEIQIA('init') را فراخوانید.
مشکلات نمایشتعارض استایل · انباشت · افزونه‌های شخص ثالث
بارگذاری‌شده اما نامرئی: معمولاً CSS سراسری سایت موقعیت حباب را بازنویسی می‌کند، یا افزونه نقشه حرارتی / تحلیل / SEO، DOM را تغییر داده و ظرف را می‌پوشاند؛ F12 → Elements، ظرف meiqia را بیابید و display:none، z-index بازنده یا مختصات بیرون صفحه را بررسی کنید.
موبایل / SDKوب موبایل · AppKey · push
وب موبایل / رایانه همان قطعه است اما استقرار جدا می‌خواهد؛ درون‌برنامه از SDK بومی استفاده می‌کند (ابتدا «افزودن پیکربندی APP» در کنسول برای AppKey)؛ push به «بدون push» (فقط درون‌برنامه) و «سرور push سفارشی» (حتی پس از خروج از برنامه تحویل) تقسیم می‌شود.
فراخوانی‌های APIwithoutBtn · showPanel · اطلاعات مشتری
دکمه خودتان؟ _MEIQIA('withoutBtn') دکمه بومی را پنهان می‌کند و هنگام کلیک برای باز کردن گفتگو _MEIQIA('showPanel') را فراخوانید؛ اطلاعات مشتری را در زمان‌بندی init ارسال / همگام کنید، وگرنه اثر ندارد.

گام ۲: نادیده‌گرفته‌شده‌ترین علت ریشه‌ای → L2 (ابزارک = تزریق JS ناهمگام شخص ثالث خارج از دامنه)

یک ایده کافی است: ابزارک وب 美洽 یک مؤلفه ایستا تعبیه‌شده در صفحه شما نیست — بلکه یک meiqia.js است که به‌صورت ناهمگام از دامنه بیرونی 美洽 بارگذاری می‌شود، سپس به‌صورت پویا یک ظرف گفتگو (DOM / iframe) تزریق می‌کند و یک اتصال طولانی‌مدت میان‌مبدأ به سرورهای 美洽 باز می‌کند. این بیشتر «ناپدیدشدن‌های مرموز» را توضیح می‌دهد: ① اسکریپت شخص ثالث خارج از دامنه است، پس AdBlock / uBlock آن را با فهرست‌های «ردیابی / تبلیغات» مسدود می‌کنند (ERR_BLOCKED_BY_CLIENT) و دکمه هرگز ظاهر نمی‌شود (هرچند کنسول سالم به‌نظر برسد)؛ ② DOM را ناهمگام تزریق می‌کند: در <head> مسدود می‌کند، تغییر مسیر SPA ظرف را نابود می‌کند و خودبه‌خود بازسازی نمی‌شود (از manualInit + _MEIQIA('init') استفاده کنید)، و افزونه‌های نقشه حرارتی / تحلیل که DOM را تغییر می‌دهند می‌توانند آن را بپوشانند؛ ③ با entId به شرکت شما پیوند می‌خورد و با فهرست سفید دامنه کنترل می‌شود: با entId اشتباه یا دامنه غیرمجاز اسکریپت بارگذاری می‌شود اما «گفتگو را وصل نمی‌کند»؛ ④ موبایل و رایانه دو یکپارچه‌سازی جدا هستند، درون‌برنامه SDK است (AppKey)، و push به «بدون push» و «سرور push سفارشی» تقسیم می‌شود. با دیدن این زنجیره تزریق، هر علامت زیر یک منطق دارد. زیر، پنل خودبررسی؛ پایین‌تر پنج روش و برآوردهای ۲۰۲۶.

موارد واقعی — خواندن سریع

  • کد چسبانده شده اما حباب گفتگو نیست: ابتدا F12 → Network، meiqia.js را جستجو کنید — نبود درخواست یا غیر ۲۰۰ یعنی جای اشتباه / گیر در حافظه پنهان؛ حافظه پنهان CDN را پاک کنید یا در حالت ناشناس دوباره باز کنید.
  • کنسول می‌گوید meiqia.js net::ERR_BLOCKED_BY_CLIENT: یک adblock (AdBlock / uBlock / AdGuard) اسکریپت گفتگوی شخص ثالث را با قوانین تبلیغات مسدود کرده — افزونه را خاموش یا سایت را به فهرست سفید بیفزایید؛ به همین دلیل «برای برخی کاربران نیست در حالی که کنسول شما سالم به‌نظر می‌رسد».
  • اسکریپت بارگذاری شده (کنسول typeof _MEIQIA تابع است) اما دکمه نیست: معمولاً CSS قالب حباب را بیرون صفحه رانده یا افزونه نقشه حرارتی / تحلیل انباشت DOM را تغییر داده — F12 → Elements، ظرف 美洽 را بیابید و مورد به مورد بررسی کنید.
  • صفحه اصلی در SPA با Vue / React گفتگو دارد اما مسیر دوم ندارد: تغییر مسیر SPA ظرف را نابود کرده و ابزارک بازسازی نشده — برای توقف مقداردهی خودکار از manualInit استفاده و در قلاب مسیر _MEIQIA('init') را برای نصب مجدد فراخوانید.
  • گفتگو باز می‌شود اما کارشناسان پیام بازدیدکننده دریافت نمی‌کنند: محتمل‌ترین حالت entId با میز کار همخوان نیست (قطعه دیگری کپی شده / حساب‌ها قاطی شده) — شناسه شرکت در تنظیمات - تیم - جستجوی شناسه را با entId در کد مقایسه کنید.
  • در staging کار می‌کند، روی دامنه تولید بارگذاری نمی‌شود: دامنه تولید از طریق «افزودن سایت یکپارچه‌سازی» مجاز نشده — دامنه زنده را به فهرست بیفزایید و دوباره منتشر کنید.
  • می‌خواهید دکمه گرد بومی 美洽 را بردارید و دکمه «تماس» خودتان را به‌کار ببرید: برای پنهان‌کردن بومی _MEIQIA('withoutBtn') را فراخوانید، سپس _MEIQIA('showPanel') را به دکمه‌تان ببندید تا گفتگو باز شود.

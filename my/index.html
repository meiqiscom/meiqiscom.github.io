美洽 ချတ် widget မပေါ်ခြင်း / ပေါင်းစပ်အမှားဖြေရှင်းခြင်း

美洽(Meiqia) code ကိုကူးထည့်ပြီးသော်လည်း ဝက်ဘ်ချတ် widget မပေါ်, ချတ်ခလုတ်ဘယ်တော့မှမပေါ်, console က meiqia.js ပိတ်ဆို့ထားကြောင်းပြ, သို့မဟုတ် agent များ chat မရ? အရင်ဆုံးအချက်သုံးချက်ရှင်းပါ: script လုံးဝ load မဖြစ်ဘူးလား (တည်နေရာ / adblock / cache), load ဖြစ်ပေမယ့် style / stacking ကြောင့်ပုန်းနေလား, ဒါမှမဟုတ် load ကောင်းပေမယ့် config မကိုက်ဘူးလား (entId / domain / SPA lifecycle)? သင့် လက္ခဏာ သို့မဟုတ် keyword ရှာပါ (ဥပမာ not showing, adblock, entId, SPA, mobile, sdk push): အရင် L1 အပြုအမူနှင့် တရားဝင်ရပ်တည်ချက်, ပြီးနောက် L2 အရင်းခံအကြောင်းရင်းနှင့် ဖြေရှင်းနည်း. ထည့်သွင်းခြင်း / လမ်းညွှန်? ကြည့်ရန် 美洽 ဒေါင်းလုဒ် / customer service ပေါင်းစပ်လမ်းညွှန်.

အဆင့် 1: script load ဖြစ်မဖြစ်ကြည့်, ပြီးနောက် လေးအုပ်စုခွဲပါ

美洽 chat မပေါ်သောအခါ 10 ကြိမ်တွင် 9 ကြိမ်က လေးအုပ်စုထဲမှ တစ်ခု; အရင် script တကယ် load ဖြစ်မဖြစ်ကြည့်ပါ: F12 → Network, meiqia.js ရှာပါ; request မရှိ သို့မဟုတ် status 200 မဟုတ်လျှင် load မဖြစ်ပါ. ① Load မအောင်မြင်: code နေရာမှား (</body> ရှေ့မှာရှိရမည်, <head> ထဲမဟုတ်), adblock စည်းမျဉ်းဖြင့်ပိတ်ဆို့ (ERR_BLOCKED_BY_CLIENT), CDN / browser cache တွင်ပိတ်မိ, သို့မဟုတ် HTTPS certificate / mixed content. ② Config / ခွင့်ပြုချက်: entId က workbench နှင့်မကိုက် (load ဖြစ်ပေမယ့် agent များ chat မရ), သို့မဟုတ် ဆိုဒ် domain ကို console တွင်ခွင့်မပြု. ③ Framework ပေါင်းစပ်: Vue/React/Next SPA route ပြောင်းချိန် container ကိုဖျက်ပြီး widget ပြန်မတည်ဆောက်. ④ ပြသခြင်း / mobile / SDK: ဆိုဒ် CSS က bubble ကို screen ပြင်ပသို့တွန်း, တတိယပါတီ plugin က DOM ဖုံး, mobile သီးသန့် deploy လို, in-app SDK လို. ပိုပြည့်စုံ: 美洽 ဝက်ဘ်ပေါင်းစပ်လမ်းညွှန်.

Load မအောင်မြင်တည်နေရာ · adblock · cache · HTTPS
美洽 JS ကို ရှေ့မှာကူးထည့် ( ထဲမဟုတ်); F12 → Network, meiqia.js ရှာပြီး status စစ် (200 ဖြစ်ရမည်); ERR_BLOCKED_BY_CLIENT များသောအားဖြင့် adblock extension — incognito တွင်စမ်း သို့မဟုတ် whitelist; ထုတ်ဝေပြီး CDN cache ရှင်း; mixed content မပါဘဲ HTTPS အပြည့်သေချာပါ.
Config / ခွင့်ပြုချက်entId · domain whitelist · sub-channel
entId သည် သင့်ကုမ္ပဏီ၏ သီးသန့် id; workbench နှင့်မကိုက်လျှင် «script load ဖြစ်ပေမယ့် agent များ chat မရ» (settings - team - ID ရှာ); ဆိုဒ် domain ကို «ပေါင်းစပ်ဆိုဒ်ထည့်ရန်» ဖြင့်ခွင့်ပြုရမည်; business line တစ်ခုစီအတွက် သီးခြား sub-channel (probe) သုံးပါ.
Framework ပေါင်းစပ်SPA · route ပြောင်း · manualInit
Vue/React/Next SPA route ပြောင်းချိန် DOM ဖျက်ပြီး widget ပြန်မတည်ဆောက်; auto-init ရပ်ရန် _MEIQIA('manualInit') သုံးပြီး ပြန် mount ရန် route hook (useEffect / mounted) တွင် _MEIQIA('init') ခေါ်ပါ.
ပြသခြင်းပြဿနာstyle ပဋိပက္ခ · stacking · တတိယပါတီ plugin
load ဖြစ်ပေမယ့်မမြင်ရ: များသောအားဖြင့် ဆိုဒ်တစ်ခုလုံး CSS က bubble တည်နေရာ override, သို့မဟုတ် heatmap / analytics / SEO plugin က DOM ပြောင်းပြီး container ဖုံး; F12 → Elements, meiqia container ရှာပြီး display:none, z-index ရှုံး, သို့မဟုတ် screen ပြင်ပ coordinate စစ်ပါ.
Mobile / SDKmobile ဝက်ဘ် · AppKey · push
mobile / PC ဝက်ဘ်က snippet တူပေမယ့် သီးခြား deploy လို; in-app က native SDK သုံး (အရင် AppKey အတွက် console တွင် «APP config ထည့်»); push က «push မရှိ» (in-app သာ) နှင့် «custom push server» (app ထွက်ပြီးနောက်လည်းရောက်) ခွဲ.
API ခေါ်ဆိုမှုwithoutBtn · showPanel · customer info
ကိုယ်ပိုင်ခလုတ်? _MEIQIA('withoutBtn') က native ခလုတ်ဖုံး, chat ဖွင့်ရန် click တွင် _MEIQIA('showPanel') ခေါ်; customer info ကို init အချိန်တွင်ပို့ / sync, မဟုတ်လျှင်အကျိုးမသက်ရောက်.

အဆင့် 2: အလျစ်လျူရှုခံရဆုံး အရင်းခံအကြောင်းရင်း → L2 (widget = တတိယပါတီ off-domain အချိန်မကိုက် JS injection)

အယူအဆတစ်ခုပင်လုံလောက်: 美洽 ဝက်ဘ် widget သည် သင့်စာမျက်နှာတွင်မြှုပ်ထားသော static component မဟုတ် — ၎င်းသည် 美洽 ၏ off-domain မှ အချိန်မကိုက် load ဖြစ်သော meiqia.js ဖြစ်ပြီး chat container (DOM / iframe) ကို dynamic ထည့်သွင်းကာ 美洽 server များသို့ ရှည်လျားသော cross-origin connection ဖွင့်သည်. ၎င်းက «ထူးဆန်းစွာပျောက်ခြင်း» အများစုကိုရှင်းပြသည်: ① ၎င်းသည် တတိယပါတီ off-domain script ဖြစ်၍ AdBlock / uBlock က «tracking / ကြော်ငြာ» စာရင်းများဖြင့်ပိတ်ဆို့ (ERR_BLOCKED_BY_CLIENT) ၍ ခလုတ်ဘယ်တော့မှမပေါ် (console ပုံမှန်ပြနေသော်လည်း); ② ၎င်းသည် DOM ကိုအချိန်မကိုက် inject လုပ်သည်: <head> တွင် ၎င်းကပိတ်ဆို့, SPA route ပြောင်းခြင်းက container ဖျက်ပြီး အလိုအလျောက်ပြန်မတည်ဆောက် (manualInit + _MEIQIA('init') သုံးပါ), DOM ပြောင်းသော heatmap / analytics plugin များက ၎င်းကိုဖုံးနိုင်; ③ ၎င်းသည် entId ဖြင့်သင့်ကုမ္ပဏီနှင့်ချိတ်ပြီး domain whitelist ဖြင့်ထိန်းချုပ်: entId မှား သို့မဟုတ် domain ခွင့်မပြုလျှင် script load ဖြစ်ပေမယ့် «chat မချိတ်»; ④ mobile နှင့် PC သည် သီးခြားပေါင်းစပ်နှစ်ခု, in-app က SDK (AppKey), push က «push မရှိ» နှင့် «custom push server» ခွဲ. ဤ injection ကွင်းဆက်ကိုမြင်ပြီးနောက် အောက်ပါလက္ခဏာတိုင်းတွင် ယုတ္တိတစ်ခုရှိ. အောက်တွင် self-check panel; အောက်ပိုင်းတွင် နည်းငါးမျိုးနှင့် 2026 ခန့်မှန်းချက်.

တကယ်ဖြစ်ရပ်များ — အမြန်ဖတ်

  • code ကူးထည့်ပြီးသော်လည်း chat bubble မရှိ: အရင် F12 → Network, meiqia.js ရှာ — request မရှိ သို့မဟုတ် non-200 ဆိုလျှင် နေရာမှား / cache ပိတ်မိ; CDN cache ရှင်း သို့မဟုတ် incognito တွင်ပြန်ဖွင့်.
  • console က meiqia.js net::ERR_BLOCKED_BY_CLIENT ဆို: adblock (AdBlock / uBlock / AdGuard) က တတိယပါတီ chat script ကို ကြော်ငြာစည်းမျဉ်းဖြင့်ပိတ်ဆို့ — extension ပိတ် သို့မဟုတ် ဆိုဒ် whitelist; «သင့် console ပုံမှန်ပြနေစဉ် အသုံးပြုသူအချို့အတွက်ပျောက်» ရခြင်းအကြောင်းရင်းပင်.
  • script load ဖြစ် (console typeof _MEIQIA က function) ပေမယ့် ခလုတ်မရှိ: များသောအားဖြင့် theme CSS က bubble ကို screen ပြင်ပတွန်း သို့မဟုတ် heatmap / analytics plugin က DOM stacking ပြောင်း — F12 → Elements, 美洽 container ရှာပြီးတစ်ခုချင်းစစ်.
  • Vue / React SPA တွင် home page မှာ chat ရှိပေမယ့် ဒုတိယ route မှာမရှိ: SPA route ပြောင်းခြင်းက container ဖျက်ပြီး widget ပြန်မတည်ဆောက် — auto-init ရပ်ရန် manualInit သုံးပြီး ပြန် mount ရန် route hook တွင် _MEIQIA('init') ခေါ်.
  • chat ဖွင့်ရပေမယ့် agent များ visitor message မရ: များသောအားဖြင့် entId က workbench နှင့်မကိုက် (သူများ snippet ကူး / account ရောထွေး) — settings - team - ID ရှာ ၏ company ID ကို code ၏ entId နှင့်တိုက်ဆိုင်ကြည့်.
  • staging တွင်အလုပ်လုပ်, production domain တွင် load မဖြစ်: production domain ကို «ပေါင်းစပ်ဆိုဒ်ထည့်ရန်» ဖြင့်ခွင့်မပြု — live domain ကို စာရင်းထဲထည့်ပြီးပြန်ထုတ်ဝေ.
  • 美洽 ၏ round native ခလုတ်ကိုဖယ်ပြီး ကိုယ်ပိုင် «ဆက်သွယ်ရန်» ခလုတ်သုံးချင်: native ဖုံးရန် _MEIQIA('withoutBtn') ခေါ်, ပြီးနောက် chat ဖွင့်ရန် ကိုယ့်ခလုတ်နှင့် _MEIQIA('showPanel') ချည်နှောင်.

美洽 أداة الدردشة لا تظهر / إصلاح خطأ التكامل

ألصقت كود 美洽(Meiqia) لكن أداة دردشة الويب لا تظهر، زر الدردشة لا يظهر أبدًا، تُبلّغ الكونسول عن حظر meiqia.js، أو الوكلاء لا يستقبلون الدردشات؟ وضّح ثلاثة أمور أولًا: هل لم يُحمّل السكربت إطلاقًا (الموضع / adblock / الذاكرة المؤقتة)، حُمِّل لكن أُخفي بالتنسيق / التكديس، أم حُمِّل جيدًا لكن الإعداد غير متطابق (entId / النطاق / دورة حياة SPA)؟ ابحث عن العَرَض أو الكلمة المفتاحية (مثل not showing، adblock، entId، SPA، mobile، sdk push): أولًا سلوك L1 والموضع الرسمي، ثم السبب الجذري L2 والحل. التثبيت / الدليل؟ انظر تنزيل 美洽 / دليل تكامل خدمة العملاء.

الخطوة 1: تحقق من تحميل السكربت ثم قسّم إلى أربع مجموعات

عندما لا تظهر دردشة 美洽، في 9 من 10 مرات تكون إحدى أربع مجموعات؛ انظر أولًا هل حُمِّل السكربت أصلًا: F12 → Network، ابحث عن meiqia.js؛ لا طلب أو حالة غير 200 يعني لم يُحمّل. ① فشل التحميل: الكود في المكان الخطأ (يجب أن يكون قبل </body>، لا في <head>)، محظور بقاعدة adblock (ERR_BLOCKED_BY_CLIENT)، محتجز بذاكرة CDN / المتصفح المؤقتة، أو شهادة HTTPS / محتوى مختلط. ② الإعداد / التفويض: entId غير متطابق مع لوحة العمل (يُحمّل لكن الوكلاء لا يستقبلون الدردشات)، أو نطاق الموقع غير مُفوَّض في الكونسول. ③ تكامل إطار العمل: تطبيق SPA من Vue/React/Next يدمّر الحاوية عند تغيير المسار ولا يُعاد بناء الأداة. ④ العرض / الجوال / SDK: CSS الموقع يدفع الفقاعة خارج الشاشة، إضافة طرف ثالث تغطي الـ DOM، الجوال يحتاج نشرًا خاصًا، داخل التطبيق يحتاج SDK. أكثر اكتمالًا: دليل تكامل ويب 美洽.

فشل التحميلالموضع · adblock · الذاكرة المؤقتة · HTTPS
ألصق JS الخاص بـ 美洽 قبل (لا في )؛ F12 → Network، ابحث عن meiqia.js وتحقق من الحالة (يجب أن تكون 200)؛ ERR_BLOCKED_BY_CLIENT عادة إضافة adblock — اختبر في وضع التصفح المتخفي أو أضف للقائمة البيضاء؛ امسح ذاكرة CDN المؤقتة بعد النشر؛ تأكّد من HTTPS كامل بلا محتوى مختلط.
الإعداد / التفويضentId · القائمة البيضاء للنطاق · القنوات الفرعية
entId هو المعرّف الفريد لشركتك؛ عند عدم تطابقه مع لوحة العمل «يُحمَّل السكربت لكن الوكلاء لا يستقبلون الدردشات» (الإعدادات - الفريق - بحث المعرّف)؛ يجب تفويض نطاق الموقع عبر «إضافة موقع تكامل»؛ استخدم قناة فرعية (مسبار) منفصلة لكل خط أعمال.
تكامل إطار العملSPA · تغيير المسار · manualInit
تطبيق SPA من Vue/React/Next يدمّر الـ DOM عند تغيير المسار ولا يُعاد بناء الأداة؛ استخدم _MEIQIA('manualInit') لإيقاف التهيئة التلقائية واستدعِ _MEIQIA('init') في خطّاف مسار (useEffect / mounted) لإعادة التركيب.
مشكلات العرضتعارض التنسيق · التكديس · إضافات الطرف الثالث
محمَّل لكن غير مرئي: عادة CSS عام للموقع يتجاوز موضع الفقاعة، أو إضافة خريطة حرارية / تحليلات / SEO تعدّل الـ DOM وتغطّي الحاوية؛ F12 → Elements، ابحث عن حاوية meiqia وتحقق من display:none، z-index خاسر، أو إحداثيات خارج الشاشة.
الجوال / SDKويب الجوال · AppKey · الدفع
ويب الجوال / الحاسوب هو نفس المقطع لكن يحتاج نشرًا خاصًا؛ داخل التطبيق يستخدم SDK أصلي (أولًا «إضافة إعداد APP» في الكونسول لـ AppKey)؛ ينقسم الدفع إلى «بلا دفع» (داخل التطبيق فقط) و«خادم دفع مخصص» (يُسلَّم حتى بعد مغادرة التطبيق).
استدعاءات APIwithoutBtn · showPanel · معلومات العميل
زرّك الخاص؟ _MEIQIA('withoutBtn') يخفي الزر الأصلي، واستدعِ _MEIQIA('showPanel') عند النقر لفتح الدردشة؛ مرّر / زامن معلومات العميل ضمن توقيت init، وإلا فلن يكون له أثر.

الخطوة 2: السبب الجذري الأكثر إغفالًا → L2 (الأداة = حقن JS غير متزامن من طرف ثالث خارج النطاق)

فكرة واحدة تكفي: أداة ويب 美洽 ليست مكوّنًا ثابتًا مضمّنًا في صفحتك — بل هي meiqia.js يُحمَّل بشكل غير متزامن من نطاق 美洽 الخارجي ثم يحقن ديناميكيًا حاوية دردشة (DOM / iframe) ويفتح اتصالًا طويل الأمد عبر الأصول إلى خوادم 美洽. هذا يفسّر معظم «الاختفاءات الغامضة»: ① إنه سكربت طرف ثالث خارج النطاق، لذا تحظره AdBlock / uBlock عبر قوائم «التتبّع / الإعلانات» (ERR_BLOCKED_BY_CLIENT) ولا يظهر الزر أبدًا (رغم أن الكونسول يبدو سليمًا)؛ ② يحقن الـ DOM بشكل غير متزامن: في <head> يحجب، تغيير مسار SPA يدمّر الحاوية ولا يُعاد بناؤها تلقائيًا (استخدم manualInit + _MEIQIA('init'))، وإضافات الخرائط الحرارية / التحليلات التي تعدّل الـ DOM قد تغطّيها؛ ③ يرتبط بشركتك عبر entId ويُقيَّد بقائمة بيضاء للنطاقات: مع entId خاطئ أو نطاق غير مفوَّض يُحمَّل السكربت لكنه «لا يوصل الدردشة»؛ ④ الجوال والحاسوب تكاملان منفصلان، داخل التطبيق هو SDK (AppKey)، والدفع ينقسم إلى «بلا دفع» مقابل «خادم دفع مخصص». بمجرد رؤية سلسلة الحقن هذه، لكل عَرَض أدناه منطق واحد. أدناه لوحة الفحص الذاتي؛ وأسفلها الطرق الخمس وتقديرات 2026.

حالات واقعية — قراءة سريعة

  • الكود ملصوق لكن لا فقاعة دردشة: أولًا F12 → Network، ابحث عن meiqia.js — لا طلب أو غير 200 يعني موضعًا خاطئًا / محتجزًا بالذاكرة المؤقتة؛ امسح ذاكرة CDN المؤقتة أو أعد الفتح في وضع التصفح المتخفي.
  • تقول الكونسول meiqia.js net::ERR_BLOCKED_BY_CLIENT: حظر adblock (AdBlock / uBlock / AdGuard) سكربت الدردشة الخارجي بقواعد الإعلانات — عطّل الإضافة أو أضف الموقع للقائمة البيضاء؛ لهذا «يختفي لبعض المستخدمين بينما تبدو كونسولك سليمة».
  • السكربت محمَّل (الكونسول typeof _MEIQIA دالة) لكن لا زر: عادة CSS القالب دفع الفقاعة خارج الشاشة أو غيّرت إضافة خريطة حرارية / تحليلات تكديس الـ DOM — F12 → Elements، ابحث عن حاوية 美洽 وافحص عنصرًا عنصرًا.
  • الصفحة الرئيسية فيها دردشة في SPA من Vue / React لكن مسار ثانٍ لا: دمّر تغيير مسار SPA الحاوية ولم يُعَد بناء الأداة — استخدم manualInit لإيقاف التهيئة التلقائية واستدعِ _MEIQIA('init') في خطّاف المسار لإعادة التركيب.
  • تُفتح الدردشة لكن الوكلاء لا يستقبلون رسائل الزوّار: الأرجح أن entId غير متطابق مع لوحة العمل (نُسخ مقطع شخص آخر / اختلطت الحسابات) — قارن معرّف الشركة في الإعدادات - الفريق - بحث المعرّف بـ entId في الكود.
  • يعمل على بيئة الاختبار، لا يُحمَّل على نطاق الإنتاج: نطاق الإنتاج غير مفوَّض عبر «إضافة موقع تكامل» — أضف النطاق الحي للقائمة ثم أعد النشر.
  • تريد إزالة زر 美洽 الأصلي الدائري واستخدام زر «تواصل» خاص بك: استدعِ _MEIQIA('withoutBtn') لإخفاء الأصلي، ثم اربط _MEIQIA('showPanel') بزرّك لفتح الدردشة.

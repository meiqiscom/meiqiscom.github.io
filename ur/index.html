美洽 چیٹ ویجٹ ظاہر نہیں ہو رہا / انٹیگریشن خرابی ٹھیک کریں

美洽(Meiqia) کوڈ چسپاں کیا مگر ویب چیٹ ویجٹ ظاہر نہیں ہو رہا، چیٹ بٹن کبھی نہیں آتا، کنسول meiqia.js بلاک بتا رہا ہے، یا ایجنٹس کو چیٹ نہیں ملتیں؟ پہلے تین باتیں واضح کریں: کیا اسکرپٹ بالکل لوڈ نہیں ہوا (مقام / adblock / کیشے)، لوڈ ہوا مگر اسٹائل / اسٹیکنگ سے چھپ گیا، یا اچھا لوڈ ہوا مگر کنفیگ مطابقت نہیں رکھتا (entId / ڈومین / SPA لائف سائیکل)؟ اپنی علامت یا کلیدی لفظ تلاش کریں (مثلاً not showing، adblock، entId، SPA، mobile، sdk push): پہلے L1 رویہ اور سرکاری مقام، پھر L2 بنیادی وجہ اور حل۔ انسٹال / گائیڈ؟ دیکھیں 美洽 ڈاؤن لوڈ / کسٹمر سروس انٹیگریشن گائیڈ۔

مرحلہ 1: دیکھیں اسکرپٹ لوڈ ہوا یا نہیں، پھر چار گروہوں میں بانٹیں

جب 美洽 چیٹ ظاہر نہ ہو، 10 میں سے 9 بار یہ چار گروہوں میں سے ایک ہوتا ہے؛ پہلے دیکھیں کیا اسکرپٹ بالکل لوڈ ہوا: F12 → Network، meiqia.js تلاش کریں؛ کوئی درخواست نہیں یا حیثیت 200 نہیں تو لوڈ نہیں ہوا۔ ① لوڈ ناکامی: کوڈ غلط جگہ (</body> سے پہلے ہونا چاہیے، <head> میں نہیں)، adblock اصول سے بلاک (ERR_BLOCKED_BY_CLIENT)، CDN / براؤزر کیشے میں پھنسا، یا HTTPS سرٹیفکیٹ / مخلوط مواد۔ ② کنفیگ / اجازت: entId ورک بینچ سے غیر مطابق (لوڈ ہوتا ہے مگر ایجنٹس کو چیٹ نہیں ملتیں)، یا سائٹ ڈومین کنسول میں مجاز نہیں۔ ③ فریم ورک انٹیگریشن: Vue/React/Next SPA روٹ تبدیلی پر کنٹینر تباہ کر دیتا ہے اور ویجٹ دوبارہ نہیں بنتا۔ ④ ڈسپلے / موبائل / SDK: سائٹ CSS ببل کو اسکرین سے باہر دھکیلتا ہے، تھرڈ پارٹی پلگ ان DOM ڈھانپتا ہے، موبائل کو الگ تعیناتی چاہیے، ان-ایپ کو SDK چاہیے۔ مزید مکمل: 美洽 ویب انٹیگریشن گائیڈ۔

لوڈ ناکامیمقام · adblock · کیشے · HTTPS
美洽 JS کو سے پہلے چسپاں کریں ( میں نہیں)؛ F12 → Network، meiqia.js تلاش کر کے حیثیت چیک کریں (200 ہونی چاہیے)؛ ERR_BLOCKED_BY_CLIENT عموماً adblock ایکسٹینشن — اِنکوگنیٹو میں جانچیں یا وائٹ لسٹ کریں؛ اشاعت کے بعد CDN کیشے صاف کریں؛ مخلوط مواد کے بغیر مکمل HTTPS یقینی بنائیں۔
کنفیگ / اجازتentId · ڈومین وائٹ لسٹ · ذیلی چینل
entId آپ کی کمپنی کی منفرد ID ہے؛ ورک بینچ سے عدم مطابقت پر «اسکرپٹ لوڈ ہوتا ہے مگر ایجنٹس کو چیٹ نہیں ملتیں» (ترتیبات - ٹیم - ID تلاش)؛ سائٹ ڈومین کو «انٹیگریشن سائٹ شامل کریں» سے مجاز کرنا ہوگا؛ ہر کاروباری لائن کے لیے الگ ذیلی چینل (پروب) استعمال کریں۔
فریم ورک انٹیگریشنSPA · روٹ تبدیلی · manualInit
Vue/React/Next SPA روٹ تبدیلی پر DOM تباہ کرتا ہے اور ویجٹ دوبارہ نہیں بنتا؛ auto-init روکنے کو _MEIQIA('manualInit') استعمال کریں اور دوبارہ ماؤنٹ کے لیے روٹ ہک (useEffect / mounted) میں _MEIQIA('init') کال کریں۔
ڈسپلے مسائلاسٹائل تصادم · اسٹیکنگ · تھرڈ پارٹی پلگ ان
لوڈ مگر نظر نہیں آتا: عموماً سائٹ-وسیع CSS ببل کا مقام اوور رائیڈ کرتا ہے، یا ہیٹ میپ / اینالیٹکس / SEO پلگ ان DOM بدل کر کنٹینر ڈھانپتا ہے؛ F12 → Elements، meiqia کنٹینر تلاش کر کے display:none، ہارا z-index، یا اسکرین سے باہر کوآرڈینیٹس چیک کریں۔
موبائل / SDKموبائل ویب · AppKey · پش
موبائل / PC ویب ایک ہی اسنپٹ ہے مگر الگ تعیناتی چاہیے؛ ان-ایپ نیٹو SDK استعمال کرتا ہے (پہلے AppKey کے لیے کنسول میں «APP کنفیگ شامل کریں»)؛ پش «کوئی پش نہیں» (صرف ان-ایپ) اور «کسٹم پش سرور» (ایپ چھوڑنے کے بعد بھی پہنچتا) میں بٹتا ہے۔
API کالزwithoutBtn · showPanel · کسٹمر معلومات
اپنا بٹن؟ _MEIQIA('withoutBtn') نیٹو بٹن چھپاتا ہے، اور چیٹ کھولنے کو کلک پر _MEIQIA('showPanel') کال کریں؛ کسٹمر معلومات init وقت میں بھیجیں / مطابق کریں، ورنہ اثر نہیں ہوگا۔

مرحلہ 2: سب سے زیادہ نظرانداز بنیادی وجہ → L2 (ویجٹ = تھرڈ پارٹی آف-ڈومین غیر متزامن JS انجیکشن)

ایک خیال کافی ہے: 美洽 ویب ویجٹ آپ کے صفحے میں سرایت شدہ جامد جزو نہیں — یہ ایک meiqia.js ہے جو 美洽 کے آف-ڈومین سے غیر متزامن لوڈ ہوتا ہے، پھر متحرک طور پر چیٹ کنٹینر (DOM / iframe) انجیکٹ کرتا ہے اور 美洽 سرورز سے طویل کراس-اوریجن کنکشن کھولتا ہے۔ یہ زیادہ تر «پراسرار غائب ہونے» کی وضاحت کرتا ہے: ① یہ تھرڈ پارٹی آف-ڈومین اسکرپٹ ہے، لہٰذا AdBlock / uBlock «ٹریکنگ / اشتہار» فہرستوں سے بلاک کرتے ہیں (ERR_BLOCKED_BY_CLIENT) اور بٹن کبھی نہیں آتا (چاہے کنسول ٹھیک لگے)؛ ② یہ DOM کو غیر متزامن انجیکٹ کرتا ہے: <head> میں یہ بلاک کرتا ہے، SPA روٹ تبدیلی کنٹینر تباہ کرتی ہے اور خود دوبارہ نہیں بنتا (manualInit + _MEIQIA('init') استعمال کریں)، اور DOM بدلنے والے ہیٹ میپ / اینالیٹکس پلگ ان اسے ڈھانپ سکتے ہیں؛ ③ یہ entId سے آپ کی کمپنی سے جڑتا ہے اور ڈومین وائٹ لسٹ سے کنٹرول ہوتا ہے: غلط entId یا غیر مجاز ڈومین پر اسکرپٹ لوڈ ہوتا ہے مگر «چیٹ کنیکٹ نہیں کرتا»؛ ④ موبائل اور PC دو الگ انٹیگریشنز ہیں، ان-ایپ SDK (AppKey) ہے، اور پش «کوئی پش نہیں» بمقابلہ «کسٹم پش سرور» میں بٹتا ہے۔ یہ انجیکشن سلسلہ دیکھنے کے بعد نیچے کی ہر علامت کا ایک منطق ہے۔ نیچے سیلف-چیک پینل؛ مزید نیچے پانچ طریقے اور 2026 تخمینے۔

  • کوڈ چسپاں کیا مگر چیٹ ببل نہیں: پہلے F12 → Network، meiqia.js تلاش کریں — کوئی درخواست نہیں یا غیر-200 یعنی غلط مقام / کیشے میں پھنسا؛ CDN کیشے صاف کریں یا اِنکوگنیٹو میں دوبارہ کھولیں۔
  • کنسول کہتا ہے meiqia.js net::ERR_BLOCKED_BY_CLIENT: ایک adblock (AdBlock / uBlock / AdGuard) نے تھرڈ پارٹی چیٹ اسکرپٹ کو اشتہار اصولوں سے بلاک کیا — ایکسٹینشن بند کریں یا سائٹ وائٹ لسٹ کریں؛ اسی لیے «کچھ صارفین کے لیے غائب جبکہ آپ کا کنسول ٹھیک لگتا ہے»۔
  • اسکرپٹ لوڈ ہوا (کنسول typeof _MEIQIA function) مگر بٹن نہیں: عموماً تھیم CSS نے ببل کو اسکرین سے باہر دھکیلا یا ہیٹ میپ / اینالیٹکس پلگ ان نے DOM اسٹیکنگ بدلی — F12 → Elements، 美洽 کنٹینر تلاش کر کے ایک ایک چیک کریں۔

  • چیٹ کھلتی ہے مگر ایجنٹس کو وزیٹر پیغامات نہیں ملتے: زیادہ امکان entId ورک بینچ سے غیر مطابق (کسی اور کا اسنپٹ کاپی / اکاؤنٹس مل گئے) — ترتیبات - ٹیم - ID تلاش کی کمپنی ID کو کوڈ کے entId سے ملائیں۔
  • اسٹیجنگ پر کام کرتا ہے، پروڈکشن ڈومین پر لوڈ نہیں ہوتا: پروڈکشن ڈومین «انٹیگریشن سائٹ شامل کریں» سے مجاز نہیں — لائیو ڈومین فہرست میں شامل کر کے دوبارہ شائع کریں۔

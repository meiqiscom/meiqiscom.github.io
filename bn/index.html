美洽 চ্যাট উইজেট দেখাচ্ছে না / ইন্টিগ্রেশন ত্রুটি ঠিক করুন

美洽(Meiqia) কোড পেস্ট করেছেন কিন্তু ওয়েব চ্যাট উইজেট দেখাচ্ছে না, চ্যাট বোতাম কখনো আসে না, কনসোল meiqia.js ব্লকড বলছে, বা এজেন্টরা চ্যাট পান না? আগে তিনটি বিষয় পরিষ্কার করুন: স্ক্রিপ্ট কি একদম লোডই হয়নি (অবস্থান / adblock / ক্যাশ), লোড হয়েছে কিন্তু স্টাইল / স্ট্যাকিং দিয়ে লুকানো, নাকি ভালো লোড হয়েছে কিন্তু কনফিগ মেলে না (entId / ডোমেন / SPA জীবনচক্র)? আপনার উপসর্গ বা কীওয়ার্ড খুঁজুন (যেমন not showing, adblock, entId, SPA, mobile, sdk push): আগে L1 আচরণ ও অফিশিয়াল অবস্থান, তারপর L2 মূল কারণ ও সমাধান। ইনস্টল / গাইড? দেখুন 美洽 ডাউনলোড / গ্রাহক সেবা ইন্টিগ্রেশন গাইড।

ধাপ ১: স্ক্রিপ্ট লোড হয়েছে কিনা দেখুন, তারপর চার দলে ভাগ করুন

美洽 চ্যাট না দেখালে ১০-এর মধ্যে ৯ বার এটি চার দলের একটি; আগে দেখুন স্ক্রিপ্ট আদৌ লোড হয়েছে কিনা: F12 → Network, meiqia.js খুঁজুন; কোনো অনুরোধ না থাকলে বা স্ট্যাটাস ২০০ না হলে লোড হয়নি। ① লোড ব্যর্থতা: কোড ভুল জায়গায় (</body>-এর আগে হওয়া উচিত, <head>-এ নয়), adblock নিয়মে ব্লকড (ERR_BLOCKED_BY_CLIENT), CDN / ব্রাউজার ক্যাশে আটকা, বা HTTPS সার্টিফিকেট / মিশ্র কনটেন্ট। ② কনফিগ / অনুমোদন: entId ওয়ার্কবেঞ্চের সাথে অমিল (লোড হয় কিন্তু এজেন্টরা চ্যাট পান না), বা সাইট ডোমেন কনসোলে অনুমোদিত নয়। ③ ফ্রেমওয়ার্ক ইন্টিগ্রেশন: Vue/React/Next SPA রুট পরিবর্তনে কন্টেইনার ধ্বংস করে এবং উইজেট পুনর্নির্মিত হয় না। ④ প্রদর্শন / মোবাইল / SDK: সাইট CSS বাবলকে স্ক্রিনের বাইরে ঠেলে দেয়, থার্ড-পার্টি প্লাগইন DOM ঢেকে দেয়, মোবাইলে নিজস্ব ডিপ্লয় লাগে, ইন-অ্যাপে SDK লাগে। আরো সম্পূর্ণ: 美洽 ওয়েব ইন্টিগ্রেশন গাইড।

লোড ব্যর্থতাঅবস্থান · adblock · ক্যাশ · HTTPS
美洽 JS -এর আগে পেস্ট করুন (-এ নয়); F12 → Network, meiqia.js খুঁজে স্ট্যাটাস যাচাই করুন (২০০ হওয়া উচিত); ERR_BLOCKED_BY_CLIENT সাধারণত adblock এক্সটেনশন — ইনকগনিটোতে পরীক্ষা বা হোয়াইটলিস্ট করুন; প্রকাশের পর CDN ক্যাশ মুছুন; মিশ্র কনটেন্ট ছাড়া পূর্ণ HTTPS নিশ্চিত করুন।
কনফিগ / অনুমোদনentId · ডোমেন হোয়াইটলিস্ট · সাব-চ্যানেল
entId আপনার কোম্পানির অনন্য আইডি; ওয়ার্কবেঞ্চের সাথে অমিলে «স্ক্রিপ্ট লোড হয় কিন্তু এজেন্টরা চ্যাট পান না» (সেটিংস - টিম - আইডি অনুসন্ধান); সাইট ডোমেন «ইন্টিগ্রেশন সাইট যোগ» দিয়ে অনুমোদন করতে হবে; প্রতি ব্যবসায়িক লাইনে আলাদা সাব-চ্যানেল (প্রোব) ব্যবহার করুন।
ফ্রেমওয়ার্ক ইন্টিগ্রেশনSPA · রুট পরিবর্তন · manualInit
Vue/React/Next SPA রুট পরিবর্তনে DOM ধ্বংস করে এবং উইজেট পুনর্নির্মিত হয় না; অটো-init থামাতে _MEIQIA('manualInit') ব্যবহার করুন এবং পুনঃমাউন্টে রুট হুকে (useEffect / mounted) _MEIQIA('init') কল করুন।
প্রদর্শন সমস্যাস্টাইল দ্বন্দ্ব · স্ট্যাকিং · থার্ড-পার্টি প্লাগইন
লোড কিন্তু অদৃশ্য: সাধারণত সাইট-ব্যাপী CSS বাবলের অবস্থান ওভাররাইড করে, বা হিটম্যাপ / অ্যানালিটিক্স / SEO প্লাগইন DOM পরিবর্তন করে কন্টেইনার ঢাকে; F12 → Elements, meiqia কন্টেইনার খুঁজে display:none, হারানো z-index, বা স্ক্রিন-বাইরের স্থানাঙ্ক যাচাই করুন।
মোবাইল / SDKমোবাইল ওয়েব · AppKey · পুশ
মোবাইল / PC ওয়েব একই স্নিপেট কিন্তু নিজস্ব ডিপ্লয় লাগে; ইন-অ্যাপ নেটিভ SDK ব্যবহার করে (প্রথমে কনসোলে AppKey-এর জন্য «APP কনফিগ যোগ»); পুশ «নো পুশ» (শুধু ইন-অ্যাপ) ও «কাস্টম পুশ সার্ভার» (অ্যাপ ছাড়ার পরও পৌঁছায়)-এ ভাগ।
API কলwithoutBtn · showPanel · গ্রাহক তথ্য
নিজের বোতাম? _MEIQIA('withoutBtn') নেটিভ বোতাম লুকায়, এবং চ্যাট খুলতে ক্লিকে _MEIQIA('showPanel') কল করুন; গ্রাহক তথ্য init সময়ের মধ্যে পাঠান / সিঙ্ক করুন, নইলে কার্যকর হবে না।

ধাপ ২: সবচেয়ে উপেক্ষিত মূল কারণ → L2 (উইজেট = থার্ড-পার্টি অফ-ডোমেন অ্যাসিনক্রোনাস JS ইনজেকশন)

একটি ধারণাই যথেষ্ট: 美洽 ওয়েব উইজেট আপনার পৃষ্ঠায় এমবেড করা স্ট্যাটিক কম্পোনেন্ট নয় — এটি একটি meiqia.js যা 美洽-এর অফ-ডোমেন থেকে অ্যাসিনক্রোনাসভাবে লোড হয়, তারপর গতিশীলভাবে একটি চ্যাট কন্টেইনার (DOM / iframe) ইনজেক্ট করে এবং 美洽 সার্ভারের সাথে দীর্ঘস্থায়ী ক্রস-অরিজিন সংযোগ খোলে। এটি বেশিরভাগ «রহস্যময় অদৃশ্য হওয়া» ব্যাখ্যা করে: ① এটি থার্ড-পার্টি অফ-ডোমেন স্ক্রিপ্ট, তাই AdBlock / uBlock «ট্র্যাকিং / বিজ্ঞাপন» তালিকা দিয়ে ব্লক করে (ERR_BLOCKED_BY_CLIENT) এবং বোতাম কখনো আসে না (কনসোল ঠিক দেখালেও); ② এটি DOM অ্যাসিনক্রোনাসভাবে ইনজেক্ট করে: <head>-এ এটি ব্লক করে, SPA রুট পরিবর্তন কন্টেইনার ধ্বংস করে এবং নিজে পুনর্নির্মিত হয় না (manualInit + _MEIQIA('init') ব্যবহার করুন), আর DOM পরিবর্তনকারী হিটম্যাপ / অ্যানালিটিক্স প্লাগইন এটি ঢেকে দিতে পারে; ③ এটি entId দিয়ে আপনার কোম্পানির সাথে বাঁধা এবং ডোমেন হোয়াইটলিস্ট দিয়ে নিয়ন্ত্রিত: ভুল entId বা অননুমোদিত ডোমেনে স্ক্রিপ্ট লোড হয় কিন্তু «চ্যাট সংযোগ করে না»; ④ মোবাইল ও PC দুটি আলাদা ইন্টিগ্রেশন, ইন-অ্যাপ হলো SDK (AppKey), আর পুশ «নো পুশ» বনাম «কাস্টম পুশ সার্ভার»-এ ভাগ। এই ইনজেকশন শৃঙ্খল দেখলে নিচের প্রতিটি উপসর্গের একটি যুক্তি থাকে। নিচে স্ব-যাচাই প্যানেল; আরো নিচে পাঁচ পদ্ধতি ও 2026 অনুমান।

মাত্রাঅনুমান / তুলনা

ইন্টিগ্রেশন আসলে কীওয়েব উইজেট = DOM ইনজেক্টকারী থার্ড-পার্টি অফ-ডোমেন অ্যাসিনক্রোনাস JS + দীর্ঘস্থায়ী ক্রস-অরিজিন সংযোগ (এমবেড স্ট্যাটিক কম্পোনেন্ট নয়); তাই অবস্থান, adblock নিয়ম, CSS স্ট্যাকিং, SPA জীবনচক্রে প্রভাবিত

বাস্তব ঘটনা — দ্রুত পড়ুন

  • কোড পেস্ট করেছেন কিন্তু চ্যাট বাবল নেই: আগে F12 → Network, meiqia.js খুঁজুন — কোনো অনুরোধ নেই বা নন-২০০ মানে ভুল অবস্থান / ক্যাশে আটকা; CDN ক্যাশ মুছুন বা ইনকগনিটোতে আবার খুলুন।
  • কনসোল বলে meiqia.js net::ERR_BLOCKED_BY_CLIENT: adblock (AdBlock / uBlock / AdGuard) থার্ড-পার্টি চ্যাট স্ক্রিপ্টকে বিজ্ঞাপন নিয়মে ব্লক করেছে — এক্সটেনশন বন্ধ বা সাইট হোয়াইটলিস্ট করুন; এজন্যই «কিছু ব্যবহারকারীর কাছে নেই অথচ আপনার কনসোল ঠিক দেখায়»।
  • স্ক্রিপ্ট লোড হয়েছে (কনসোল typeof _MEIQIA function) কিন্তু বোতাম নেই: সাধারণত থিম CSS বাবলকে স্ক্রিনের বাইরে ঠেলেছে বা হিটম্যাপ / অ্যানালিটিক্স প্লাগইন DOM স্ট্যাকিং বদলেছে — F12 → Elements, 美洽 কন্টেইনার খুঁজে এক এক করে যাচাই করুন।
  • Vue / React SPA-তে হোম পেজে চ্যাট আছে কিন্তু দ্বিতীয় রুটে নেই: SPA রুট পরিবর্তন কন্টেইনার ধ্বংস করেছে এবং উইজেট পুনর্নির্মিত হয়নি — অটো-init থামাতে manualInit ব্যবহার করুন এবং পুনঃমাউন্টে রুট হুকে _MEIQIA('init') কল করুন।
  • চ্যাট খোলে কিন্তু এজেন্টরা দর্শকের বার্তা পান না: সম্ভবত entId ওয়ার্কবেঞ্চের সাথে অমিল (অন্যের স্নিপেট কপি / অ্যাকাউন্ট মিশে গেছে) — সেটিংস - টিম - আইডি অনুসন্ধানের কোম্পানি আইডি কোডের entId-এর সাথে মেলান।
  • স্টেজিংয়ে কাজ করে, প্রোডাকশন ডোমেনে লোড হয় না: প্রোডাকশন ডোমেন «ইন্টিগ্রেশন সাইট যোগ» দিয়ে অনুমোদিত নয় — লাইভ ডোমেন তালিকায় যোগ করে আবার প্রকাশ করুন।
  • 美洽-এর গোল নেটিভ বোতাম সরিয়ে নিজের «যোগাযোগ» বোতাম ব্যবহার করতে চান: নেটিভ লুকাতে _MEIQIA('withoutBtn') কল করুন, তারপর চ্যাট খুলতে নিজের বোতামে _MEIQIA('showPanel') বাঁধুন।

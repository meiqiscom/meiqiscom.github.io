បានបិទភ្ជាប់កូដ 美洽(Meiqia) ប៉ុន្តែ widget ឆាតគេហទំព័រ មិនបង្ហាញ, ប៊ូតុងឆាតមិនដែលលេច, console រាយការណ៍ថា meiqia.js ត្រូវបានទប់ស្កាត់, ឬ ភ្នាក់ងារមិនទទួលឆាត? បញ្ជាក់បីរឿងជាមុនសិន: តើ script មិនបានផ្ទុកទាល់តែសោះ (ទីតាំង / adblock / cache), ផ្ទុកប៉ុន្តែលាក់ដោយរចនាបថ / ការដាក់ត្រួត, ឬផ្ទុកល្អប៉ុន្តែ config មិនត្រូវគ្នា (entId / domain / វដ្តជីវិត SPA)? ស្វែងរក រោគសញ្ញា ឬពាក្យគន្លឹះ (ឧ. not showing, adblock, entId, SPA, mobile, sdk push): ដំបូង ឥរិយាបថ L1 និងទីតាំងផ្លូវការ, បន្ទាប់មក មូលហេតុឫស L2 និងដំណោះស្រាយ. ដំឡើង / មគ្គុទ្ទេសក៍? មើល ទាញយក 美洽 / មគ្គុទ្ទេសក៍រួមបញ្ចូលសេវាអតិថិជន.

ជំហាន 1: ពិនិត្យថា script ផ្ទុកឬអត់, បន្ទាប់មកបែងចែកជាបួនក្រុម

ពេល 美洽 ឆាតមិនបង្ហាញ, 9 ក្នុង 10 ដងវាជាមួយក្នុងបួនក្រុម; មើលជាមុនថា script ផ្ទុកពិតឬអត់: F12 → Network, ស្វែងរក meiqia.js; គ្មានសំណើ ឬស្ថានភាពមិនមែន 200 មានន័យថាមិនបានផ្ទុក. ① បរាជ័យផ្ទុក: កូដខុសកន្លែង (ត្រូវនៅមុន </body>, មិនមែនក្នុង <head>), ត្រូវច្បាប់ adblock ទប់ស្កាត់ (ERR_BLOCKED_BY_CLIENT), ជាប់ក្នុង cache CDN / browser, ឬវិញ្ញាបនបត្រ HTTPS / មាតិកាចម្រុះ. ② Config / ការអនុញ្ញាត: entId មិនត្រូវនឹង workbench (ផ្ទុកប៉ុន្តែភ្នាក់ងារមិនទទួលឆាត), ឬ domain គេហទំព័រមិនបានអនុញ្ញាតក្នុង console. ③ ការរួមបញ្ចូល framework: Vue/React/Next SPA បំផ្លាញ container ពេលផ្លាស់ route ហើយ widget មិនត្រូវបានបង្កើតឡើងវិញ. ④ ការបង្ហាញ / ទូរស័ព្ទ / SDK: CSS គេហទំព័រច្រានពពុះចេញក្រៅអេក្រង់, plugin ភាគីទីបីគ្របលើ DOM, ទូរស័ព្ទត្រូវ deploy ផ្ទាល់, ក្នុង app ត្រូវ SDK. ពេញលេញជាង: មគ្គុទ្ទេសក៍រួមបញ្ចូលគេហទំព័រ 美洽.

បរាជ័យផ្ទុកទីតាំង · adblock · cache · HTTPS
បិទភ្ជាប់ JS 美洽 មុន (មិនមែនក្នុង ); F12 → Network, ស្វែងរក meiqia.js ហើយពិនិត្យស្ថានភាព (គួរ 200); ERR_BLOCKED_BY_CLIENT ជាញឹកញាប់ផ្នែកបន្ថែម adblock — សាកក្នុងរបៀបអនាមិក ឬ whitelist; សម្អាត cache CDN បន្ទាប់ពីបោះពុម្ព; ធានា HTTPS ពេញដោយគ្មានមាតិកាចម្រុះ.

ផ្ទុកប៉ុន្តែមើលមិនឃើញ: ជាញឹកញាប់ CSS ទូទាំងគេហទំព័រ override ទីតាំងពពុះ, ឬ plugin heatmap / analytics / SEO កែ DOM ហើយគ្រប container; F12 → Elements, ស្វែងរក container meiqia ហើយពិនិត្យ display:none, z-index ចាញ់, ឬកូអរដោនេក្រៅអេក្រង់.

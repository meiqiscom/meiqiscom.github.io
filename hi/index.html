美洽 चैट विजेट नहीं दिख रहा / इंटीग्रेशन त्रुटि ठीक करें

美洽(Meiqia) कोड चिपकाया पर वेब चैट विजेट नहीं दिख रहा, चैट बटन कभी नहीं आता, कंसोल meiqia.js ब्लॉक बताता है, या एजेंट चैट नहीं पाते? पहले तीन बातें स्पष्ट करें: क्या स्क्रिप्ट लोड ही नहीं हुई (प्लेसमेंट / adblock / कैश), लोड हुई पर स्टाइल / स्टैकिंग से छिप गई, या ठीक लोड हुई पर कॉन्फ़िग मेल नहीं खाता (entId / डोमेन / SPA जीवनचक्र)? अपना लक्षण या कीवर्ड खोजें (जैसे not showing, adblock, entId, SPA, mobile, sdk push): पहले L1 व्यवहार व आधिकारिक स्थिति, फिर L2 मूल कारण व समाधान। इंस्टॉल / गाइड? देखें 美洽 डाउनलोड / ग्राहक सेवा इंटीग्रेशन गाइड।

चरण 1: जाँचें कि स्क्रिप्ट लोड हुई या नहीं, फिर चार समूहों में बाँटें

जब 美洽 चैट नहीं दिखता, 10 में से 9 बार यह चार समूहों में से एक होता है; पहले देखें कि स्क्रिप्ट लोड भी हुई या नहीं: F12 → Network, meiqia.js खोजें; कोई अनुरोध न हो या स्थिति 200 न हो तो लोड नहीं हुई। ① लोड विफलता: कोड गलत जगह (</body> से पहले होना चाहिए, <head> में नहीं), adblock नियम से ब्लॉक (ERR_BLOCKED_BY_CLIENT), CDN / ब्राउज़र कैश में अटका, या HTTPS प्रमाणपत्र / मिश्रित सामग्री। ② कॉन्फ़िग / प्राधिकरण: entId वर्कबेंच से मेल नहीं खाता (लोड होता है पर एजेंट चैट नहीं पाते), या साइट डोमेन कंसोल में अधिकृत नहीं। ③ फ़्रेमवर्क इंटीग्रेशन: Vue/React/Next SPA रूट बदलने पर कंटेनर नष्ट कर देता है और विजेट दोबारा नहीं बनता। ④ प्रदर्शन / मोबाइल / SDK: साइट CSS बबल को स्क्रीन से बाहर धकेलता है, थर्ड-पार्टी प्लगइन DOM को ढक देता है, मोबाइल को अपनी तैनाती चाहिए, इन-ऐप को SDK चाहिए। अधिक पूर्ण: 美洽 वेब इंटीग्रेशन गाइड।

लोड विफलताप्लेसमेंट · adblock · कैश · HTTPS
美洽 JS को से पहले चिपकाएँ ( में नहीं); F12 → Network, meiqia.js खोजें और स्थिति जाँचें (200 होनी चाहिए); ERR_BLOCKED_BY_CLIENT आमतौर पर adblock एक्सटेंशन — गुप्त मोड में जाँचें या व्हाइटलिस्ट करें; प्रकाशन के बाद CDN कैश साफ़ करें; मिश्रित सामग्री के बिना पूर्ण HTTPS सुनिश्चित करें।
कॉन्फ़िग / प्राधिकरणentId · डोमेन व्हाइटलिस्ट · उप-चैनल
entId आपकी कंपनी की अद्वितीय आईडी है; वर्कबेंच से मेल न खाने पर «स्क्रिप्ट लोड होती है पर एजेंट चैट नहीं पाते» (सेटिंग्स - टीम - आईडी खोज); साइट डोमेन «इंटीग्रेशन साइट जोड़ें» से अधिकृत होना चाहिए; प्रति व्यवसाय-लाइन अलग उप-चैनल (प्रोब) उपयोग करें।
फ़्रेमवर्क इंटीग्रेशनSPA · रूट बदलाव · manualInit
Vue/React/Next SPA रूट बदलने पर DOM नष्ट करता है और विजेट दोबारा नहीं बनता; ऑटो-इनिट रोकने हेतु _MEIQIA('manualInit') उपयोग करें और दोबारा माउंट हेतु रूट हुक (useEffect / mounted) में _MEIQIA('init') कॉल करें।
प्रदर्शन समस्याएँस्टाइल टकराव · स्टैकिंग · थर्ड-पार्टी प्लगइन
लोड पर अदृश्य: आमतौर पर साइट-व्यापी CSS बबल स्थिति को ओवरराइड करता है, या हीटमैप / एनालिटिक्स / SEO प्लगइन DOM बदलकर कंटेनर ढक देता है; F12 → Elements, meiqia कंटेनर खोजें और display:none, हारा z-index या स्क्रीन-बाहर निर्देशांक जाँचें।
मोबाइल / SDKमोबाइल वेब · AppKey · पुश
मोबाइल / PC वेब एक ही स्निपेट है पर अपनी तैनाती चाहिए; इन-ऐप नेटिव SDK उपयोग करता है (पहले कंसोल में AppKey हेतु «APP कॉन्फ़िग जोड़ें»); पुश «कोई पुश नहीं» (केवल इन-ऐप) और «कस्टम पुश सर्वर» (ऐप छोड़ने पर भी पहुँचता) में बँटता है।
API कॉलwithoutBtn · showPanel · ग्राहक जानकारी
अपना बटन? _MEIQIA('withoutBtn') नेटिव बटन छिपाता है, और चैट खोलने हेतु क्लिक पर _MEIQIA('showPanel') कॉल करें; ग्राहक जानकारी init समय के भीतर भेजें / सिंक करें, वरना प्रभावी नहीं होगा।

चरण 2: सबसे अधिक अनदेखा किया गया मूल कारण → L2 (विजेट = थर्ड-पार्टी ऑफ़-डोमेन असिंक्रोनस JS इंजेक्शन)

एक विचार काफ़ी है: 美洽 वेब विजेट आपके पेज में एम्बेडेड स्थिर घटक नहीं है — यह एक meiqia.js है जो 美洽 के ऑफ़-डोमेन स्रोत से असिंक्रोनस लोड होता है, फिर गतिशील रूप से चैट कंटेनर (DOM / iframe) इंजेक्ट करता है और 美洽 सर्वरों से लंबे समय का क्रॉस-ओरिजिन कनेक्शन खोलता है। यह अधिकांश «रहस्यमय गायब होने» को समझाता है: ① यह थर्ड-पार्टी ऑफ़-डोमेन स्क्रिप्ट है, इसलिए AdBlock / uBlock इसे «ट्रैकिंग / विज्ञापन» सूचियों से ब्लॉक करते हैं (ERR_BLOCKED_BY_CLIENT) और बटन कभी नहीं आता (कंसोल ठीक दिखने पर भी); ② यह DOM को असिंक्रोनस इंजेक्ट करता है: <head> में यह ब्लॉक करता है, SPA रूट बदलने पर कंटेनर नष्ट होता है और स्वयं दोबारा नहीं बनता (manualInit + _MEIQIA('init') उपयोग करें), और DOM बदलने वाले हीटमैप / एनालिटिक्स प्लगइन इसे ढक सकते हैं; ③ यह entId से आपकी कंपनी से बँधा है और डोमेन व्हाइटलिस्ट से नियंत्रित है: गलत entId या अनधिकृत डोमेन पर स्क्रिप्ट लोड होती है पर «चैट कनेक्ट नहीं करती»; ④ मोबाइल और PC दो अलग इंटीग्रेशन हैं, इन-ऐप SDK है (AppKey), और पुश «कोई पुश नहीं» बनाम «कस्टम पुश सर्वर» में बँटता है। इस इंजेक्शन शृंखला को देखने के बाद नीचे के हर लक्षण का एक तर्क है। नीचे स्व-जाँच पैनल; और नीचे पाँच तरीके व 2026 अनुमान।

वास्तविक मामले — त्वरित पढ़ें

  • कोड चिपकाया पर कोई चैट बबल नहीं: पहले F12 → Network, meiqia.js खोजें — कोई अनुरोध न हो या गैर-200 तो गलत जगह / कैश में अटका; CDN कैश साफ़ करें या गुप्त मोड में दोबारा खोलें।
  • कंसोल कहता है meiqia.js net::ERR_BLOCKED_BY_CLIENT: adblock (AdBlock / uBlock / AdGuard) ने थर्ड-पार्टी चैट स्क्रिप्ट को विज्ञापन नियमों से ब्लॉक किया — एक्सटेंशन बंद करें या साइट व्हाइटलिस्ट करें; इसीलिए «कुछ उपयोगकर्ताओं के लिए गायब जबकि आपका कंसोल ठीक दिखता है»।
  • स्क्रिप्ट लोड हुई (कंसोल typeof _MEIQIA function है) पर कोई बटन नहीं: आमतौर पर थीम CSS ने बबल को स्क्रीन से बाहर धकेला या हीटमैप / एनालिटिक्स प्लगइन ने DOM स्टैकिंग बदली — F12 → Elements, 美洽 कंटेनर खोजें और एक-एक जाँचें।
  • Vue / React SPA में होम पेज पर चैट है पर दूसरे रूट पर नहीं: SPA रूट बदलाव ने कंटेनर नष्ट किया और विजेट दोबारा नहीं बना — ऑटो-इनिट रोकने हेतु manualInit उपयोग करें और दोबारा माउंट हेतु रूट हुक में _MEIQIA('init') कॉल करें।
  • चैट खुलती है पर एजेंट विज़िटर संदेश नहीं पाते: सबसे संभावित entId वर्कबेंच से मेल नहीं खाता (किसी और का स्निपेट कॉपी / खाते मिल गए) — सेटिंग्स - टीम - आईडी खोज की कंपनी आईडी को कोड के entId से मिलाएँ।
  • स्टेजिंग पर काम करता है, प्रोडक्शन डोमेन पर लोड नहीं होता: प्रोडक्शन डोमेन «इंटीग्रेशन साइट जोड़ें» से अधिकृत नहीं — लाइव डोमेन सूची में जोड़ें और दोबारा प्रकाशित करें।
  • 美洽 के गोल नेटिव बटन को हटाकर अपना «संपर्क» बटन उपयोग करना चाहते हैं: नेटिव छिपाने हेतु _MEIQIA('withoutBtn') कॉल करें, फिर चैट खोलने हेतु अपने बटन से _MEIQIA('showPanel') बाँधें।
